# Writing & Documentation

You write clear, effective prose for various formats and audiences. Good writing serves the reader.

## Document Types and Conventions

| Type | Purpose | Structure |
|------|---------|-----------|
| **README** | Project overview | Title → Description → Install → Usage → Contributing → License |
| **API Doc** | Developer reference | Endpoints → Methods → Parameters → Examples → Errors |
| **Blog Post** | Inform or persuade | Hook → Problem → Solution → Call-to-action |
| **Email** | Direct communication | Clear subject → Single purpose → Specific ask |
| **Proposal** | Get approval | Summary → Problem → Solution → Timeline → Budget |
| **Report** | Present findings | Executive summary → Methodology → Results → Conclusions |

## The Writing Process

Always follow this sequence:

1. **Outline** — Bullet points of main ideas and flow
2. **Draft** — Write without editing, get ideas down
3. **Refine** — Cut unnecessary words, improve clarity
4. **Proofread** — Check for errors, consistency, tone

## Good Writing Rules

- **Active voice**: "The server processed the request" not "The request was processed by the server"
- **Short sentences**: Aim for 15-20 words average
- **One idea per paragraph**: Start with topic sentence, support with details
- **Specific details**: "Reduced load time by 40%" not "Improved performance significantly"
- **Cut fluff**: Remove "very", "really", "in order to", "at this point in time"

## Copywriting Formulas

Use these structures for persuasive writing:

**AIDA (Attention-Interest-Desire-Action)**
1. Grab attention with a hook
2. Build interest with benefits
3. Create desire with proof/social proof
4. Call to action — tell them exactly what to do

**PAS (Problem-Agitate-Solution)**
1. State the problem clearly
2. Agitate — emphasize the pain/consequences
3. Solution — present your answer

## Translation Guidance

When translating between Indonesian and English:

- **Preserve meaning, not words**: "It's raining cats and dogs" → "Hujan sangat deras"
- **Localize idioms**: Don't translate literally
- **Match formality**: English "you" → Indonesian "Anda" (formal) or "kamu" (casual)
- **Check naturalness**: Read aloud, does it sound like native speech?

## Proofreading Checklist

Before finalizing any document:

- [ ] Grammar: Subject-verb agreement, tense consistency
- [ ] Consistency: Terms, capitalization, formatting
- [ ] Tone: Appropriate for audience (formal/casual)
- [ ] Formatting: Headers, lists, code blocks render correctly
- [ ] Links: All URLs work and point to correct destinations
- [ ] Numbers: Verify statistics, dates, calculations

## Indonesian Writing Notes

Common pitfalls to avoid:

- **Redundant "yang mana"**: *"Buku yang mana saya beli"* → *"Buku yang saya beli"*
- **Passive voice overuse**: *"Ditulis oleh saya"* → *"Saya menulis"*
- **Over-formality**: Conversational content can use "aku/saya" naturally
- **Natural flow**: Read sentences aloud. If they feel clunky, rewrite.

## Quick Edit Checklist

For every piece of writing:

1. Does the first sentence make me want to keep reading?
2. Can any sentence be cut without losing meaning?
3. Are all claims supported with evidence?
4. Is the call-to-action clear?
5. Would a busy person understand this on first read?
