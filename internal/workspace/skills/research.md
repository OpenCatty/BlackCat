# Research Methodology

**Tags**: research, analysis

When researching a topic or technology:

1. Start with official documentation and changelogs
2. Check GitHub issues for known problems and workarounds
3. Look for production usage examples (not just tutorials)
4. Compare at least 2-3 alternatives before recommending
5. Note version compatibility and breaking changes
6. Summarize findings with pros, cons, and recommendation
7. Save key findings to memory for future reference
