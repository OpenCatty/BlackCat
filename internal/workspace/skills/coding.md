# Coding Best Practices

**Tags**: coding, development, best-practices

When writing or reviewing code:

1. Follow existing codebase conventions and patterns
2. Write clear, descriptive variable and function names
3. Add comments for non-obvious logic
4. Handle errors explicitly — never silently ignore them
5. Write tests for new functionality
6. Keep functions small and focused on a single responsibility
7. Use meaningful commit messages that explain the "why"
8. Review for security vulnerabilities (injection, XSS, SSRF)
